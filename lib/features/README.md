# Features

Папка содержит все функциональные модули приложения.

## Структура

```
features/
├── auth/                        # 🔐 Аутентификация
│   ├── auth_screen.dart         # Главный экран аутентификации
│   ├── auth_viewmodel.dart      # ViewModel главного экрана
│   ├── auth_state_provider.dart # Глобальное состояние аутентификации
│   ├── login/                   # Функциональность входа
│   ├── registration/            # Функциональность регистрации
│   ├── verification/            # Функциональность верификации
│   ├── success/                 # Функциональность экрана успеха
│   └── README.md                # Документация аутентификации
├── splash/                      # 🌟 Экран загрузки
│   ├── splash_screen.dart       # UI экрана загрузки
│   ├── splash_viewmodel.dart    # Бизнес-логика экрана загрузки
│   └── README.md               # Документация splash
├── onboarding/                  # 📚 Онбординг
│   ├── onboarding_screen.dart  # UI экрана онбординга
│   ├── onboarding_viewmodel.dart # Бизнес-логика онбординга
│   └── README.md               # Документация onboarding
├── home/                        # 🏠 Главный экран
│   └── main_screen.dart        # UI главного экрана
└── README.md                    # Этот файл
```

## Общий поток приложения

### 1. Splash Screen 🌟
- Первый экран при запуске
- Определяет следующий экран на основе сохраненных данных
- Показывает логотип и индикатор загрузки

### 2. Onboarding 📚
- Вводный экран для новых пользователей
- 3 страницы с описанием функциональности
- Показывается только при первом запуске

### 3. Authentication 🔐
- **Auth Screen** - выбор между регистрацией и входом
- **Registration** - создание нового аккаунта
- **Login** - вход в существующий аккаунт
- **Verification** - подтверждение кода
- **Success** - приветственное сообщение

### 4. Home 🏠
- Главный экран приложения
- Доступен после успешной аутентификации

## Архитектурные принципы

### Общие для всех модулей:
- **MVVM** паттерн
- **Riverpod** для управления состоянием
- **GoRouter** для навигации
- **Clean Architecture** с разделением на слои
- **Dependency Injection** через провайдеры

### Структура каждого модуля:
- `*_screen.dart` - UI компонент
- `*_viewmodel.dart` - Бизнес-логика
- `README.md` - Документация модуля

## Навигация

Все переходы между экранами осуществляются через `NavigationService`:
- `navigateToSplash()` - переход к экрану загрузки
- `navigateToOnboarding()` - переход к онбордингу
- `navigateToAuth()` - переход к аутентификации
- `navigateToHome()` - переход в главный экран

## Состояние приложения

Глобальное состояние управляется через:
- `auth_state_provider.dart` - состояние аутентификации
- `localStorageRepository` - локальные данные пользователя
- `navigationService` - навигационное состояние
